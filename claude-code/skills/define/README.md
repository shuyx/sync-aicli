# Define Skill - 概念定义和解释助手

## 快速开始

这个 skill 帮助你建立一个个人的**概念知识库**。每当你有概念性问题时，只需告诉我，我会：

1. ✅ **清晰解释** - 用合理的篇幅解释这个概念
2. 📝 **自动保存** - 将解释保存到 markdown 文件中
3. 🏷️ **自动命名** - 用你提问中的概念名称作为文件名
4. 📚 **建立知识库** - 所有概念自动保存到你的 Obsidian 库中

## 使用方式

### 基础用法

只需问概念性问题：

```
你好，什么是"强化学习"？
能不能解释一下"Domain Randomization"？
"Sim-to-Real" 是什么意思？
"UMI" 有什么用途？
"Mobile ALOHA" 与 "UMI" 有什么区别？
```

### 支持的问题类型

| 问题类型 | 例子 |
|---------|------|
| 基本定义 | 什么是 "XXX"？|
| 功能用途 | "XXX" 有什么用？|
| 应用场景 | "XXX" 能应用在哪里？|
| 价值意义 | "XXX" 的价值是什么？|
| 对比分析 | "XXX" 和 "YYY" 有什么区别？|
| 深层理解 | 能解释一下 "XXX" 吗？|

## 输出说明

### 对话中的解释

会给你一个**详细的清晰解释**（篇幅不受限制），根据概念的复杂程度尽可能详细，包括：

- **定义**: 核心概念（可多段落）
- **特性**: 关键特征（详细列表）
- **应用**: 实际用途和场景（具体例子）
- **对比**: 与相似概念的区别（详细对比）
- **相关资源**: 参考链接或相关概念
- **深入分析**: 理论基础、工作原理等

### 自动保存的文档

文件会自动保存到：
```
~/Obsidian/Obsidian Work/概念库/[概念名称].md
```

文档结构：
```markdown
# 概念名称

> 定义日期：2026-01-17
> 来源：通过 Droid define skill 生成

## 摘要（约 500 字）
第一段用 500 字左右清晰说明本文档的整体内容：
- 概念的基本定义
- 核心特性和关键点
- 主要应用场景
- 与其他概念的关系
- 文档涵盖的全部内容

让读者可以通过读摘要快速了解整个文档讲了什么。

## 基本定义
[详细的定义和说明，可多段落]

## 核心特性
[主要特性列表，可详细]

## 应用场景
[实际应用例子，具体案例]

## 优缺点对比
[与相似概念的详细对比]

## 深入分析
[理论基础、工作原理等]

## 相关概念
[相关的其他概念]

## 参考资源
[相关论文、文档等]
```

## 配置

默认配置自动使用：

```yaml
存储位置: ~/Obsidian/Obsidian Work/概念库
语言支持: 中文、英文、混合
模板类型: 完整版（full）
```

## 示例

### 例1：简单概念

**你问**：什么是 "UMI"？

**我会**：
- 第一段（摘要）：用 500 字说清楚 UMI 是什么、有什么特点、怎么用、与其他方案的区别等核心内容
- 后续段落：详细解释定义、特性、应用场景、对比分析等
- 保存到：`概念库/UMI.md`

### 例2：对比分析

**你问**："Domain Randomization" vs "System Identification"，有什么区别？

**我会**：
- 分别为两个概念生成详细文档
- 每个文档的第一段摘要（约 500 字）清晰说明该概念的定义、原理、优缺点、应用场景等
- 在摘要中说明与另一个概念的关键区别
- 分别保存两个文件

### 例3：深层理解

**你问**：能详细解释 "Sim-to-Real" 的挑战吗？

**我会**：
- 第一段摘要（约 500 字）：完整说明 Sim-to-Real 的定义、主要挑战、已有解决方案、实际应用等全貌
- 后续详细分析：深入讲解每个挑战、解决方案的原理、成功案例、最佳实践等
- 保存到：`概念库/Sim-to-Real.md`

## 最佳实践

1. **用引号标记概念** - 用 `"` 或 `` ` `` 或 **【】** 标记概念
2. **一个概念为主** - 一个问题专注一个概念最佳
3. **提供背景** - 如果是专业领域的概念，可以简单说明背景
4. **定期复习** - 建立的知识库定期复习，不断深化理解

## 常见问题

**Q: 一次可以问多个概念吗？**  
A: 可以，但每个概念会单独保存文件。建议一个问题聚焦一个概念，效果最好。

**Q: 概念定义会更新吗？**  
A: 如果你再次提问同一个概念，会生成更新的版本，覆盖旧文件。

**Q: 能改变存储位置吗？**  
A: 可以！只需告诉我新位置，我会更新配置。

**Q: 支持混合语言吗？**  
A: 完全支持！中文、英文、或者混合都没问题。
