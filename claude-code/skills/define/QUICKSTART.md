## 🎯 如何使用

### 方式 1：使用 Skill 命令

在与我（Droid）的对话中，使用 Skill 工具调用 define：

```bash
❯ 什么是 "UMI"？
```

或者显式调用：

```bash
❯ /skill define
```

只需问任何概念性问题，系统会自动识别：


## 💡 使用技巧

### 提问时的标记方式

系统会自动识别以下标记中的概念：

| 标记方式 | 例子 |
|---------|------|
| 双引号 | `什么是 "UMI"？` |
| 反引号 | `` 什么是 `强化学习`？ `` |
| 方括号 | `什么是 【Sim-to-Real】？` |
| 粗体 | `什么是 **Domain Randomization**？` |

### 获得更详细的解释

如果第一次的解释不够详细，可以追问：

```
你: 什么是 "UMI"？
我: [基本解释] + 保存到文件

你: 能详细解释一下 "UMI" 的成本优势吗？
我: [更详细的分析] + 更新保存的文件
```

### 对比多个概念

```
你: "Mobile ALOHA" vs "UMI" vs "单臂机器人"，各有什么优缺点？
我: [三方对比] + 分别保存三个文件
```

## 📚 查阅已保存的概念

### 在 Obsidian 中查看

1. 打开 Obsidian
2. 导航到 `概念库` 文件夹
3. 查看对应的 `.md` 文件

### 快速打开

在 Obsidian 中：
- **快速打开**: Cmd+O (Mac) / Ctrl+O (Win)
- **全局搜索**: Cmd+Shift+F / Ctrl+Shift+F
- **关键字搜索**: Cmd+F / Ctrl+F

### 在其他笔记中引用

在任何笔记中，使用 Obsidian 链接语法引用：

```markdown
我提到过 [[UMI]] 这个工具...

关于 [[Sim-to-Real]] 的挑战，我已经详细分析过...
```

## ❓ 常见问题

### Q: 我想重新定义某个概念，怎么做？

**A**: 只需再次提问同一个概念，新的定义会覆盖旧文件：

```
你: 什么是 "UMI"？  (第二次)
我: [新的解释，覆盖旧定义]
```

### Q: 能删除某个概念吗？

**A**: 可以直接在 Obsidian 中删除对应的 `.md` 文件。

### Q: 能导出所有概念吗？

**A**: 可以！Obsidian 支持多种导出格式：
- PDF
- HTML
- 其他格式

### Q: 中英文混合支持吗？

**A**: 完全支持！你可以用中文提问、英文提问或混合提问。

### Q: 概念库能同步到云端吗？

**A**: 可以！将 `~/Obsidian/Obsidian Work/` 目录用云同步工具（如 iCloud、Dropbox 等）同步即可。
