# Define Skill - 文档格式更新

**更新日期**: 2026-01-17  
**更新内容**: 解释性文档现在包含 500 字摘要段

## 📝 新的文档格式

每个概念定义文档现在包含：

### 第一段：摘要（约 500 字）✨

这是最重要的一段，用 500 字左右**清晰说明本文档的整体内容**，包括：

- ✅ 概念的基本定义
- ✅ 核心特性和关键点  
- ✅ 主要应用场景
- ✅ 与其他概念的关系
- ✅ 文档涵盖的全部内容

**目的**：让读者仅通过读摘要就能快速了解整个文档讲了什么

### 后续部分：详细内容

然后是其他标准部分：

- **基本定义** - 详细的定义和说明
- **核心特性** - 主要特性和特点
- **应用场景** - 实际应用例子和案例
- **优缺点对比** - 与相似概念的详细对比
- **深入分析** - 理论基础和工作原理
- **相关概念** - 相关的其他概念
- **参考资源** - 论文、文档等

## 📄 文档结构示例

```markdown
# 概念名称

> 定义日期：2026-01-17
> 来源：通过 Droid define skill 生成

## 摘要

[用 500 字说清楚：这个概念是什么、有什么特点、怎么用、与其他的关系、
本文档包含的内容等，让读者一读摘要就对整个概念有全面了解]

## 基本定义

[详细的定义...]

## 核心特性

[特性列表...]

## 应用场景

[应用例子...]

[其他部分...]
```

## 💡 摘要内容示例

### 例 1：简单工具概念

**UMI 的摘要（约 500 字）可能包括：**
- UMI 是什么（定义）
- 采集方式（手持夹爪）
- 成本极低的原因
- 主要特点（高效、快速）
- 适用场景（双臂操作采集）
- 与 Mobile ALOHA 的区别
- 与其他采集方式的对比
- 适合谁使用

### 例 2：复杂技术概念

**Sim-to-Real 的摘要（约 500 字）可能包括：**
- Sim-to-Real 的定义和重要性
- 核心挑战是什么
- 主要解决方案有哪些
- 不同方案的优缺点概览
- 实际应用成功案例
- 当前的最佳实践
- 文档后续会详细讲解什么

### 例 3：方法对比概念

**Domain Randomization 的摘要（约 500 字）可能包括：**
- Domain Randomization 的定义
- 核心原理简述
- 主要优势
- 主要劣势
- 与 System Identification 的关键区别
- 什么时候用 Domain Randomization
- 成功案例
- 文档后续的详细内容

有了这个摘要设计，你的概念库变得：
