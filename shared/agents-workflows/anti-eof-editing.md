# Anti-EOF Chunked Editing Strategy

## 适用范围
仅针对**文件编辑操作**（write_to_file / replace_file_content / multi_replace_file_content）。
IDE 对话框内的文字回复是流式传输的，不受此限制。

## 强制规则

### 1. 单次回复最多并行编辑 3 个文件
超过则拆分成多个步骤执行。

### 2. 单次编辑体量控制（按文件类型区分）

| 文件类型 | 单文件上限 | 精确替换上限 | 说明 |
|----------|-----------|-------------|------|
| 代码文件 | 150 行 | 80 行 | .py/.js/.ts/.go 等 |
| 文档文件 | 250 行 | 150 行 | .md/.txt/.tex 等 |
| 混合操作 | 总量 ~400 行 | - | 代码+文档混合时按总量控 |

- 优先使用 `replace_file_content` 做精确替换，而非 `write_to_file` 整文件覆写
- 超大文档（>250行）拆成多次追加或分文件

### 3. 多文件任务必须先列出计划
- 列出所有需要修改的文件
- 分批（每批 2-3 个文件）
- 逐批执行，每批之间汇报进度

### 4. 失败不重试，进一步拆分
如果编辑失败或连接中断：
- 说明当前编辑粒度仍然过大
- 将失败的编辑拆成更小的部分再执行
- **绝不以相同规模重试**
- 已成功编辑的文件无需回滚，直接从失败点继续
