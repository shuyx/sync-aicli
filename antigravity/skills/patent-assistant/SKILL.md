# Patent Assistant 专利助手（v2 融合升级版）

帮助研发人员将技术方案转化为结构化的专利技术交底书，并进行高阶的多源平台专利检索与查新分析。

## 一、功能概览

本 Skill 提供四项核心能力：

1. **结构化交底书生成**：遵循《专利审查指南》规范，支持两段式权利要求撰写、先 Claims 后 Description 的专业写作顺序
2. **多源平台专利检索（脚本模式）**：调用 `scripts/patent_search.py`，支持 Google Patents, CNIPA, Innojoy, 百度学术等多平台并行检索
3. **Obsidian Clippings 人机协同检索（推荐模式）**：Agent 生成检索式 → 用户国知局手动查询 → Web Clipper 剪藏 → Agent 威胁度评分
4. **预审规则自检**：11 条自动检查规则（6 文档 + 5 附图）+ 5 维度权利要求弱点分析

## 三、交底书生成流程

### 3.1 信息收集（四大要素）

当用户信息不足时，必须先确认：

1. **技术痛点**：现有技术存在什么缺陷？
2. **核心方案**：你的方案的核心机制/架构是什么？
3. **有益效果**：性能指标（时间、成本、精度）有什么量化提升？
4. **实施例**：至少一个具体的物理实施步骤/场景

### 3.2 撰写顺序（先 Claims 后 Description）

```
Step 1: 权利要求书 → Step 2: 发明内容 → Step 3: 具体实施方式(≥10000字)
→ Step 4: 背景技术 → Step 5: 摘要 → Step 6: 附图说明
```

### 3.3 权利要求两段式写法规范

**独立权利要求**：

```
一种[场景]的[方法/装置/系统]，包括[前序部分：共有特征]，
其特征在于：[特征部分：区别特征]
```

- 前序部分：主题名称 + 与最接近现有技术的共有必要技术特征
- 特征部分："其特征在于" + 区别于现有技术的技术特征
- 划界针对**一篇**最接近的现有技术进行

**从属权利要求**：保护范围必须落在所引用权利要求范围之内

### 3.4 交底书模板

```markdown
# 专利技术交底书

## 一、发明名称
一种[场景]的[方法/装置/系统]（≤25字）

## 二、技术领域
本发明涉及[大领域]技术领域，具体涉及[细分领域]。

## 三、背景技术
3.1 现有技术描述
3.2 现有技术缺陷：(1)... (2)... (3)...
3.3 参考文献

## 四、发明内容
4.1 技术问题（具体，避免笼统）
4.2 技术方案（用语与权利要求一致）
4.3 技术参数（量化指标）
4.4 有益效果（量化数据）

## 五、具体实施方式（≥ 10000 字）
5.1 系统总体架构
5.2 核心模块详述
5.3 实施例一（完整步骤+参数）
5.4 实施例二（变体方案）
5.5 实验数据与对比

## 六、附图说明
建议绘制：架构图、流程图、时序图...

## 七、权利要求书
（独立权利要求 + 从属权利要求）

## 八、关键词
[检索复用关键词列表]
```

## 四、预审规则自检

### 4.1 文档预审（6 条）

| 规则 | 检查项 | 严重度 |
|------|-------|--------|
| PRE001 | 发明名称格式正确且≤25字 | ERROR |
| PRE002 | 技术领域含大领域+细分领域 | WARNING |
| PRE003 | 背景技术不包含对本发明的描述 | ERROR |
| PRE004 | 发明内容含问题+方案+效果 | ERROR |
| PRE005 | 权利要求≥1独立+≥2从属 | ERROR |
| PRE006 | 附图引用与正文一致 | WARNING |

### 4.2 权利要求弱点分析（5 维度）

| 维度 | 检查内容 |
|------|---------|
| Claim Scope | 保护范围是否被写窄？含非必要限定？ |
| 从属层次 | 层级是否合理？有效扩展保护？ |
| 功能性限定 | 是否有 Means-plus-function 表述？ |
| Prior Art 暴露 | 是否容易被在先专利攻破？ |
| 术语一致性 | 全文同一概念是否只用一种叫法？ |

### 4.3 硬性质量标准

- 具体实施方式 ≥ 10000 字
- 全文术语统一
- 技术方案"范围化"描述（非单一实施例）
- 充分公开：本领域技术人员无需创造性劳动可实现

## 五、多平台检索标准流程

### 5.1 提取检索关键词

从技术方案中提取：
- 核心技术术语
- 同义词/近义词
- IPC 分类号

**常用 IPC**：

| 领域 | IPC |
|------|-----|
| 电量测量/计量 | G01R |
| 计算机软件 | G06F |
| 人工智能 | G06N |
| 电力系统 | H02J |
| 通信 | H04L, H04W |
| 物联网 | H04L 67/ |

### 5.2 执行检索脚本

```bash
# 多平台并行（最推荐）
python scripts/patent_search.py "关键词" -s all -p

# 带相似度分析
python scripts/patent_search.py "关键词" -s all -p -a

# 指定国内平台
python scripts/patent_search.py "关键词" -s cnipa,innojoy,baidu
```

### 5.3 分析结果

1. **高威胁警告**：标题/摘要高度吻合的需标红，给出规避建议
2. **Top 5 结构化列表**：平台 | 专利号 | 标题 | 摘要核心点
3. **保护范围防缩水建议**

## 六、重要原则

1. **术语统一**：全文同一事物只能有一种叫法（最重要）
2. **充分公开 vs 技术秘密**：核心方案必须充分公开，非必要细节可保留
3. **不能按功能写**：应按步骤/层次写，避免功能性描述导致核心不突出
4. **查新仅供参考**：不可代替国知局官方代理机构的查新报告
5. **国内项目必须查国内库**：`-s cnipa,innojoy`，不能只查 Google
