# 核心写作规范模块

本模块用于控制论文写作质量，重点是去AI化表达与可提交的Markdown排版。

## 一、输出排版规范（新增强制项）

### 1.1 Markdown正文规范

1. 正文默认不使用加粗和斜体。
2. 段落之间必须空一行。
3. 正文段落优先连续叙述，不用项目符号堆叠观点。
4. 同一段尽量保持单一中心观点。
5. 避免把一个完整观点拆成过多短段，能在一段内交代清楚时不强行分段。

### 1.2 允许使用列表的场景

仅以下场景可用列表：
- 计划文档（`plan/*.md`）
- 检查清单
- 参数配置
- 操作步骤

论文正文（章节正文）默认不用列表，除非用户明确要求。

### 1.3 快速排版自检

编辑 `.md` 后执行人工自检：
1. 是否出现无意义 `**加粗**`。
2. 段间是否做到“一段一空行”。
3. 是否把正文写成了条目罗列。

## 二、去AI化语言规范

### 2.1 禁用表达

1. 机械过渡词：首先、其次、最后、此外、另外、接下来。
2. 强调句式：值得注意的是、需要指出的是、重要的是、必须强调的是。
3. 空洞修饰词：非常、极其、十分、相当（无数据支撑时）。
4. 主观引导句：我认为、我觉得、我个人看法是（论文正文默认禁用）。

### 2.2 推荐表达

1. 用语义衔接替代模板衔接。
2. 用数据和事实替代形容词。
3. 长短句交替，避免等长句连续出现。
4. 用“本文”“实验结果表明”“数据说明”等客观主语组织论证。

### 2.3 句法与信息密度

1. 列表转段落时补足主语、谓语和连接成分，禁止生硬拼接条目句。
2. 一句话只承担一个核心动作，避免空泛总括句连续出现。
3. 尽量保留方法、条件、对象和数据，不用“很多”“较大提升”等模糊表述替代事实。

## 三、段落构建规则

一个标准段落建议包含：
1. 主题句：本段核心结论。
2. 支撑句：依据、证据、解释。
3. 收束句：过渡或小结。

建议长度：
- 中文正文：150-300字。
- 英文正文：100-200词。

## 四、列表转段落规则

### 错误写法

```text
本研究贡献如下：
- 提出新方法
- 完成自动化流程
- 验证有效性
```

### 推荐写法

```text
本研究提出了一种新方法，并将其整合为可执行的自动化流程。实验结果显示，该方法在目标任务上具有稳定增益，验证了其可行性与应用价值。
```

## 五、引用与事实

1. 不编造文献或数据。
2. 引用格式与全文统一（GB/T、APA、IEEE、MLA 任选其一并固定）。
3. 含有结论性表述时，优先给出处或数据。

## 六、三轮质量检查

### 第一轮：结构检查

1. 是否存在正文列表化。
2. 段落是否围绕单一中心。
3. 章节逻辑是否连续。

### 第二轮：语言检查

1. 是否出现禁用过渡词。
2. 是否出现禁用强调句式。
3. 是否存在无信息量形容词堆积。
4. 是否出现主观化表述或句子成分缺失。

### 第三轮：排版检查

1. 是否有无意义加粗。
2. 段间是否统一空一行。
3. 标点与中英文空格是否一致。

## 七、执行约束

1. 长任务开始前必须读取或创建 `plan/`。
2. 每次完成写作后，更新 `plan/progress.md` 与 `plan/notes.md`。
3. 若用户明确要求保留特定风格，以用户要求优先。
