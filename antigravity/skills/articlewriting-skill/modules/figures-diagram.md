# 流程图/结构图生成模块

本模块用于识别流程图、结构图、概念图等需求，并生成专业的画图提示词供生图AI使用。

---

## 一、适用场景

### 1.1 本模块处理的图表类型

| 图表类型 | 描述 | 示例 |
|----------|------|------|
| **流程图** | 展示步骤、流程、算法 | 数据处理流程、实验步骤 |
| **架构图** | 展示系统、模型结构 | 神经网络架构、系统架构 |
| **概念图** | 展示概念关系 | 理论框架、知识结构 |
| **组织图** | 展示层级关系 | 分类体系、组织结构 |
| **时间线** | 展示时间顺序 | 研究历程、发展阶段 |
| **对比图** | 展示差异对比 | 方法对比、前后对比 |

### 1.2 不适用本模块的图表

以下图表应使用 `figures-python.md` 模块：
- 折线图、柱状图、饼图等数据图表
- 散点图、热力图等统计图表
- 需要精确数值的任何图表

---

## 二、工作流程

```
用户需求 → 分析图表类型 → 确认是流程/结构图 → 生成提示词 → 推荐生图AI
```

### 2.1 需求分析

**判断标准：**
- 是否需要展示流程/步骤？→ 流程图
- 是否需要展示结构/组成？→ 架构图
- 是否需要展示概念关系？→ 概念图
- 是否需要精确数值？→ 转到Python模块

### 2.2 信息收集

生成提示词前，需要与用户确认：
1. **图表目的**：要表达什么内容？
2. **核心元素**：包含哪些主要部分？
3. **关系类型**：元素之间是什么关系？
4. **风格偏好**：学术风格/简约风格/彩色/黑白？
5. **语言要求**：中文/英文标注？

## 四、推荐生图AI

### 4.1 推荐工具

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Nano Banana** | 学术图表专业 | 流程图、架构图 |
| **Midjourney** | 艺术感强 | 概念图、封面图 |
| **DALL-E 3** | 理解力强 | 复杂结构图 |
| **Stable Diffusion** | 可本地部署 | 批量生成 |

### 4.2 使用建议

1. **首选 Nano Banana**：专为学术图表设计，输出最专业
2. **复杂结构用 DALL-E 3**：对复杂描述理解更好
3. **需要多次迭代用 Midjourney**：支持变体生成

## 五、示例提示词

### 示例1：数据处理流程图

```
绘制一个学术论文级别的流程图，展示"时间序列数据预处理流程"。

【整体布局】
- 从上到下垂直流动
- 白色背景，专业学术风格
- 适合IEEE期刊
- 分辨率：450 DPI

【流程步骤】

步骤1：原始数据输入
- 蓝色矩形框
- 内容："原始时间序列数据 X ∈ ℝ^(N×T)"
- 标注：N=样本数，T=时间步长

步骤2：缺失值处理
- 浅蓝色矩形框
- 内容："线性插值填充缺失值"
- 箭头从步骤1指向此处

步骤3：异常值检测
- 橙色矩形框（核心步骤）
- 内容："3σ原则检测异常点"
- 加粗边框突出

步骤4：数据标准化
- 绿色矩形框
- 内容："Z-score标准化"
- 公式：x' = (x - μ) / σ

步骤5：输出清洗数据
- 深绿色矩形框
- 内容："清洗后数据 X' ∈ ℝ^(N×T)"

【配色方案】
- 蓝色(#4E79A7)：输入/输出
- 橙色(#F28E2B)：核心处理（加粗边框）
- 绿色(#59A14F)：标准化步骤

【风格要求】
- 圆角矩形框
- 步骤间用带箭头连线
- 每个步骤标注编号
- 数学公式清晰可读
- 中文标注
```

### 示例2：Transformer架构图

```
绘制一个学术论文级别的神经网络架构图，展示"改进的Transformer编码器"结构。

【整体布局】
- 从下到上垂直流动
- 白色背景
- 适合NeurIPS/ICML风格
- 分辨率：450 DPI

【网络结构】

底部输入：
- "Input Embedding"
- 维度标注：(B, T, d_model)

模块1：位置编码
- "Positional Encoding"
- 用加号符号与输入相加

模块2：多头注意力（核心创新）
- "Multi-Head Attention"
- 红色虚线框标注"改进部分"
- 内部显示：Q, K, V 三个分支
- 标注：4 heads, d_k = 64

模块3：Add & Norm
- 残差连接（虚线绕过）
- Layer Normalization

模块4：前馈网络
- "Feed Forward"
- 标注：FFN(x) = ReLU(xW1)W2

模块5：Add & Norm
- 同模块3

顶部输出：
- "Encoder Output"
- 维度标注：(B, T, d_model)

【配色方案】
- 蓝色(#4E79A7)：注意力相关
- 绿色(#59A14F)：前馈网络
- 灰色(#6C757D)：归一化层
- 红色虚线：创新标注

【标注要求】
- 每层标注名称
- 关键维度变化标注
- 创新点用红色虚线框
- 残差连接用虚线箭头
```

## 六、对话流程

### 用户请求流程图时

**AI响应模板：**

```
我来帮你生成流程图的提示词。先确认几个信息：

1. **图表目的**：这个流程图要展示什么？
2. **主要步骤**：包含哪些关键步骤？（请列出）
3. **步骤关系**：是线性流程还是有分支/循环？
4. **语言要求**：中文还是英文标注？
5. **目标期刊**：投稿到什么类型的期刊？

请提供以上信息，我会生成专业的画图提示词。
```

### 生成提示词后

**AI响应模板：**

```
已生成画图提示词，请复制到生图AI使用：

---
[提示词内容]
---

**推荐使用**：Nano Banana（学术图表专业）

**使用步骤**：
1. 复制上述提示词
2. 打开 Nano Banana / 其他生图AI
3. 粘贴提示词并生成
4. 如需调整，告诉我具体修改需求

生成后如果需要微调，可以告诉我哪里需要修改。
```
