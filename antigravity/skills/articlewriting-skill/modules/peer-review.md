# 同行评审与批判性思维

本模块提供论文评审、自我审视和批判性分析的完整指南。

---

## 一、同行评审工作流程

### 阶段1：初步评估

**目标**：快速把握论文核心，形成初步印象。

**关键问题**：
- 核心研究问题或假设是什么？
- 主要发现和结论是什么？
- 工作是否科学合理且有意义？
- 是否适合目标期刊/会议？
- 是否存在明显的重大缺陷？

**输出**：2-3句话的简要总结，捕捉论文本质和初步印象。

### 阶段2：逐节详细审查

#### 摘要与标题
- **准确性**：摘要是否准确反映研究内容和结论？
- **清晰度**：标题是否具体、准确、信息丰富？
- **完整性**：关键发现和方法是否得到适当总结？
- **可读性**：摘要是否对广泛的科学读者可理解？

#### 引言
- **背景**：背景信息是否充分且最新？
- **理由**：研究问题是否有明确的动机和理由？
- **新颖性**：工作的原创性和重要性是否清楚阐述？
- **文献**：相关先前研究是否被适当引用？
- **目标**：研究目的/假设是否清楚陈述？

#### 方法
- **可重复性**：其他研究者能否根据描述复现研究？
- **严谨性**：方法是否适合解决研究问题？
- **细节**：协议、试剂、设备和参数是否充分描述？
- **伦理**：伦理批准、知情同意和数据处理是否正确记录？
- **统计**：统计方法是否适当、清晰描述并有理由？

**需验证的关键要素**：
- 样本量和功效分析
- 随机化和盲法程序
- 纳入/排除标准
- 数据收集协议
- 计算方法和软件版本
- 统计检验和多重比较校正

#### 结果
- **呈现**：结果是否逻辑清晰地呈现？
- **图表**：可视化是否适当、清晰且正确标注？
- **统计**：统计结果是否正确报告（效应量、置信区间、p值）？
- **客观性**：结果是否客观呈现而无过度解读？
- **完整性**：是否包含所有相关结果，包括阴性结果？

**常见问题识别**：
- 选择性报告结果
- 不适当的统计检验
- 缺少误差棒或变异性度量
- 过拟合或循环分析
- 批次效应或混杂变量
- 缺少对照或验证实验

#### 讨论
- **解释**：结论是否有数据支持？
- **局限性**：研究局限性是否被承认和讨论？
- **背景**：发现是否适当地置于现有文献中？
- **推测**：推测是否与数据支持的结论明确区分？
- **意义**：影响和重要性是否清楚阐述？

**警示信号**：
- 过度陈述的结论
- 忽视矛盾证据
- 从相关数据做因果声明
- 对局限性讨论不足
- 没有机制证据的机制声明

### 阶段3：方法论和统计严谨性

**统计评估**：
- 统计假设是否满足（正态性、独立性、方差齐性）？
- 是否报告效应量和p值？
- 是否适当应用多重检验校正？
- 是否提供置信区间？
- 样本量是否有功效分析支持？
- 参数与非参数检验选择是否适当？
- 缺失数据处理是否正确？

**实验设计**：
- 对照是否适当和充分？
- 重复是否足够（生物学和技术重复）？
- 潜在混杂因素是否被识别和控制？
- 随机化是否正确实施？
- 盲法程序是否充分？

### 阶段4：可重复性和透明度

**数据可用性**：
- 原始数据是否存放在适当的存储库？
- 是否提供公共数据库的登录号？
- 数据共享限制是否有理由（如患者隐私）？

**代码和材料**：
- 分析代码是否可用（GitHub、Zenodo等）？
- 独特材料是否可获得或有足够描述以便重建？

**报告标准**：
- 论文是否遵循学科特定的报告指南（CONSORT、PRISMA、ARRIVE等）？

## 二、审稿报告结构

### 总结陈述

提供简洁的整体评估（1-2段）：
- 研究的简要概述
- 总体建议（接受、小修、大修、拒稿）
- 关键优点（2-3个要点）
- 关键缺点（2-3个要点）
- 对重要性和合理性的底线评估

### 主要意见

列出显著影响论文有效性、可解释性或重要性的关键问题。按顺序编号以便参考。

**主要意见通常包括**：
- 基本方法论缺陷
- 不适当的统计分析
- 不支持或过度陈述的结论
- 缺少关键对照或实验
- 严重的可重复性问题
- 文献覆盖的重大空白
- 伦理问题

**每个主要意见应**：
1. 清楚陈述问题
2. 解释为什么有问题
3. 建议具体解决方案或额外实验
4. 指出解决是否对发表至关重要

### 次要意见

列出不太关键但会改善清晰度、完整性或呈现的问题。

**次要意见通常包括**：
- 不清晰的图标签或图例
- 缺少方法细节
- 排版或语法错误
- 改进数据呈现的建议
- 轻微统计报告问题
- 加强结论的补充分析请求

## 三、批判性思维框架

### 偏倚检测

#### 认知偏倚（研究者）
- **确认偏倚**：是否只强调支持性发现？
- **HARKing**：假设是事先陈述还是看到结果后形成？
- **发表偏倚**：文献中是否缺少阴性结果？
- **挑选数据**：证据是否被选择性报告？

#### 选择偏倚
- **抽样偏倚**：样本是否代表目标人群？
- **志愿者偏倚**：参与者是否以系统方式自我选择？
- **流失偏倚**：组间退出是否有差异？
- **幸存者偏倚**：样本中是否只有"幸存者"可见？

#### 测量偏倚
- **观察者偏倚**：期望是否可能影响观察？
- **回忆偏倚**：回顾性报告是否系统性不准确？
- **社会期望**：反应是否偏向可接受性？
- **工具偏倚**：测量工具是否系统性出错？

#### 分析偏倚
- **P-hacking**：是否进行多次分析直到出现显著性？
- **结果切换**：非显著结果是否被显著结果替换？
- **选择性报告**：是否报告所有计划的分析？
- **亚组挖掘**：亚组分析是否在没有校正的情况下进行？

### 逻辑谬误识别

#### 因果谬误
- **事后归因**："B跟在A后面，所以A导致了B"
- **相关=因果**：混淆关联与因果
- **反向因果**：误将原因当结果
- **单一原因谬误**：将复杂结果归因于一个因素

#### 泛化谬误
- **草率泛化**：从小样本得出广泛结论
- **轶事谬误**：个人故事作为证明
- **挑选数据**：只选择支持性证据
- **生态谬误**：将群体模式应用于个体

#### 统计谬误
- **基础率忽视**：忽略先验概率
- **德州神枪手**：在随机数据中发现模式
- **多重比较**：未校正多重检验
- **检察官谬误**：混淆P(E|H)与P(H|E)

## 四、证据质量评估

### 研究设计层级

1. 系统综述/荟萃分析（干预效果最高）
2. 随机对照试验
3. 队列研究
4. 病例对照研究
5. 横断面研究
6. 病例系列/报告
7. 专家意见（最低）

**重要**：高层级设计不一定质量更好。设计良好的观察性研究可能比执行不佳的RCT更强。

### GRADE考虑因素

**降级因素**：
- 偏倚风险
- 研究间不一致性
- 间接性（错误的人群/干预/结果）
- 不精确性（宽置信区间、小样本）
- 发表偏倚

**升级因素**：
- 大效应量
- 剂量-反应关系
- 混杂因素会减少（而非增加）效应

### 证据收敛

**更强的情况**：
- 多个独立重复
- 不同研究组和环境
- 不同方法论收敛于相同结论
- 机制和实证证据一致

**更弱的情况**：
- 单一研究或研究组
- 文献中有矛盾发现
- 明显的发表偏倚
- 无重复尝试

## 五、自我审稿提示词

### 投稿前自查

```markdown
# Role
你是一位以严苛、精准著称的资深学术审稿人，熟悉计算机科学领域顶级会议的评审标准。

# Task
请深入阅读并分析我的论文。基于我指定的【投稿目标】，撰写一份严厉但具有建设性的审稿报告。

# Constraints
1. 评审基调（严苛模式）：
   - 默认态度：请抱着拒稿的预设心态进行审查
   - 拒绝客套：省略所有无关痛痒的赞美，直接切入核心缺陷

2. 审查维度：
   - 原创性：该工作是实质性的突破还是边际增量？
   - 严谨性：数学推导是否有跳跃？实验对比是否公平？消融实验是否充分？
   - 一致性：引言中声称的贡献在实验部分是否真的得到了验证？

3. 输出格式：
   - Part 1 [Review Report]：
     * Summary: 一句话总结文章核心
     * Strengths: 1-2点真正有价值的贡献
     * Weaknesses (Critical): 3-5个可能导致拒稿的致命问题
     * Rating: 预估评分（1-10分）
   - Part 2 [Strategic Advice]：
     * 直击痛点：解释Critical Weaknesses因何而起
     * 行动指南：具体建议该补什么实验、该重写哪段逻辑

# Input
投稿目标：[例如：ICML 2026]
论文内容：[粘贴论文或上传PDF]
```

### 快速质量检查

```markdown
# Role
你是论文质量检查员。

# Task
请快速检查以下论文内容是否存在以下问题：

1. **逻辑一致性**：引言的声明是否在实验中得到验证？
2. **术语一致性**：核心概念是否在全文保持一致命名？
3. **数据支持**：所有结论是否有实验数据支持？
4. **对照完整性**：是否与足够的baseline进行了比较？
5. **消融充分性**：是否验证了每个关键模块的贡献？

# Output
- 如果没有问题：[检测通过]
- 如果有问题：分点列出，每点包含位置和具体问题

# Input
[粘贴论文内容]
```

## 六、审稿语气与方法

### 最佳实践

- **建设性**：将批评框架为改进机会
- **具体**：提供具体例子和可操作建议
- **平衡**：承认优点和缺点
- **尊重**：记住作者投入了大量努力
- **客观**：关注科学，而非科学家
- **彻底**：不忽视问题，但适当优先排序
- **清晰**：避免模糊或含糊的批评

### 避免事项

- 人身攻击或轻蔑语言
- 讽刺或居高临下
- 没有具体例子的模糊批评
- 要求超出范围的不必要实验
- 要求遵循个人偏好而非最佳实践
- 在双盲评审中暴露身份

## 七、不同论文类型的特殊考虑

### 原创研究文章
- 强调严谨性、可重复性和新颖性
- 评估重要性和影响
- 验证结论是数据驱动的
- 检查完整的方法和适当的对照

### 综述和荟萃分析
- 评估文献覆盖的全面性
- 评估搜索策略和纳入/排除标准
- 验证系统方法和无偏倚
- 检查批判性分析而非仅仅总结
- 对于荟萃分析，评估统计方法和异质性

### 方法论文
- 强调验证和与现有方法的比较
- 评估协议/代码的可重复性和可用性
- 评估相对现有方法的改进
- 检查实施的足够细节

### 会议论文
- 考虑页数限制
- 关注核心贡献的清晰度
- 验证主要声明有足够支持
- 检查是否符合会议主题

## 八、审稿检查清单

在完成审稿前，验证：

- [ ] 总结陈述清楚传达整体评估
- [ ] 主要问题被清楚识别和论证
- [ ] 建议的修改具体且可操作
- [ ] 次要问题被注意但正确分类
- [ ] 统计方法已被评估
- [ ] 可重复性和数据可用性已评估
- [ ] 伦理考虑已验证
- [ ] 图表已评估质量和完整性
- [ ] 写作质量已评估
- [ ] 全文语气建设性且专业
- [ ] 审稿彻底但与论文范围相称
- [ ] 建议与识别的问题一致
